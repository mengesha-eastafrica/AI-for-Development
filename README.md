# AI for Development

## Purpose
This repository serves as an operational sandbox and prompt-engineering library exploring the safe, structured integration of Artificial Intelligence (AI) and Large Language Models (LLMs) into international development workflows. It prioritizes data privacy, contextual localization, and strict text sanitization to safely augment technical output without compromising institutional data security.

## Repository Taxonomy

*   **`AI-for-MEL/`** • Prompts and workflows designed for synthesizing vast monitoring datasets, indicator mapping, and automated logframe verification.
*   **`AI-for-Evaluation/`** • Structured prompt matrices for evaluating report thematic consistency, coding qualitative text, and extracting key lessons learned.
*   **`AI-for-Proposals/`** • Bid-preparation workflows, compliance matrix parsing frameworks, and past-performance alignment structures.
*   **`AI-for-Research/`** • Prompt protocols for secondary literature reviews, contextual synthesis, and legal empowerment pathway mapping.
*   **`Prompt-Library/`** • System prompts and safe parameter configurations for development professionals interacting with generative models.

## 🔒 Professional Protocol: The Anonymization & Data Security Framework

To leverage generative models safely within international consulting workflows, all operations must adhere to this data governance protocol before text processing:

### 1. Data Masking Mandatory Rules
*   **Personally Identifiable Information (PII):** All explicit staff names, donor contact details, and precise geographical community indicators must be replaced with tokenized variables (e.g., `[Consultant_ID_01]`, `[Target_District_A]`).
*   **Financial Records:** Proprietary facility financial statements or granular budget items must be transformed into percentage distributions or scalable parametric cost formulas.
*   **Donor Constraints:** Any text extracted from donor-restricted, confidential reports or proprietary evaluations must be stripped of its descriptive context, keeping only the underlying methodological engine or structural outline.

### 2. Prompt Architecture Standards
All prompts stored in this repository utilize structured markdown headers, role-based definitions, explicit constraint parameters, and few-shot examples to ensure high-density, contextually precise outputs from target language models.
